上流工程の初期段階では、**ヒアリング内容**がまだ整理されていない状態から、**ユースケース**・**機能要件**・**非機能要件**などを導き出す必要があります。ここでは、まとまっていない**ヒアリングメモ**を例に、**Copilot** を使って情報を整理・分類する流れを紹介します。

---

## ヒアリングメモ（サンプル・まだまとまりきっていない）

下記のメモは「ユーザーが気軽にメッセージのやり取りができるアプリを開発する」というテーマで、  
**クライアントとのやり取り**を抜粋したものとします。

```
- 今回のアプリは、ユーザーどうしでメッセージを送れたらいい。無料でOK
- 友達を登録してチャット、あと公式アカウントみたいなのも将来ほしいかも
- なんかスタンプ的なもの？ 画像送信？ んー最初はテキストだけにするかも
- 通知がくるようにしてほしいって話だったはず
- スマホメイン。PCはあとでいいかも
- すぐリリースできる形にしたい（2〜3か月ぐらい？）
- 稼働初期はそんなにアクセス多くないと思う。せいぜい数百ユーザー？
- でもUI可愛いと嬉しいんだけど… あ、それはデザイナーと相談か
- サーバーはAWSでいいよね？コストはなるべく安く
- グループチャットやビデオ通話は後々検討
- ユーザー: 若い世代がメインかも(10~30代)
- 広告…いれたいかもしれない。どうしよう
- セキュリティは常識的にHTTPS、パスワード保存には気をつけて
- あとユーザーが自分のプロフィール編集できたらいいね
- 当面は日本語のみ対応かな
- 長期的に海外向けもありか
```

**ポイント**: 情報がバラバラに書かれており、まだ“ビジネス要件 / 機能 / 非機能” として整理されていない。

## 1. ユースケースの作成

### Copilot の活用イメージ

- **Copilot** に「このヒアリングメモから、ユーザーや管理者それぞれのユースケース（使い方のシナリオ）をざっくり書いて」と頼む  
- ユーザー視点（一般ユーザー / 将来の公式アカウントユーザー）や管理者視点で、それぞれの行動パターンを**ユースケース形式**で生成

#### プロンプト例

```text
このヒアリングメモを元に、ユーザー（一般ユーザー）と運営（管理者）それぞれのユースケースを3つずつ書いてください。
使い方シナリオをできるだけ具体的に。
```

##### 出力例

```
【一般ユーザーのユースケース】
1. 「友達を登録してテキストチャット」
   - ユーザーはアプリを開き、友達のIDを検索して追加 → チャット画面でメッセージ送信
2. 「通知を受け取る」
   - ユーザーがオフラインでも、友達からメッセージが来ればプッシュ通知が届く
3. 「プロフィール編集」
   - ユーザーがアイコン画像やニックネームを設定

【管理者のユースケース】
1. 「公式アカウントの準備」
   - 管理者が将来的に公式アカウントを作成するための仕組みを管理画面で設定
2. 「システムの使用状況を把握」
   - 同時アクセス数や負荷を確認
3. 「広告設定 (将来的)」
   - 管理者が広告枠を設定したり、レイアウトを切り替える
```

## 2. 機能要件の作成

### Copilot の活用イメージ

- ユースケースの次は「**どんな機能が必要か**」を**箇条書き**で洗い出し  
- ヒアリングメモ＋ユースケースを基に、Copilot に**機能要件**をまとめてもらう

#### プロンプト例

```text
今度は機能要件をリストアップしてください。上記のユースケースとヒアリングメモを参考に。
項目はできるだけ詳細にお願いします。
```

##### 出力例

```
[機能要件]
1. ユーザー管理
   - 新規登録/ログイン (最低限の認証)
   - プロフィール編集 (アイコン画像, ユーザー名)

2. フレンドリスト機能
   - ユーザーIDで検索, 追加/削除
   - フレンド一覧の取得

3. チャット送受信
   - 個別チャット (テキストメッセージのみ)
   - プッシュ通知 (メッセージ受信時に端末へ通知)
   - 将来的にスタンプ/画像送信を追加検討

4. 管理者機能 (将来)
   - 公式アカウント作成/管理
   - 広告設定管理 (未定)
   - システム負荷状況をモニタリング

5. 多言語対応 (将来)
   - 現在は日本語のみ
   - 英語ほかへの対応見込み
```

## 3. 非機能要件の作成

### Copilot の活用イメージ

- 「**どの程度の同時アクセスに耐えるか**」「**応答速度はどれくらい**」「**セキュリティ水準**」などを**Copilot** にまとめてもらう  
- Copilot が**ヒアリングメモ**から拾い上げ、**追加のアイディア**も出してくれる

#### プロンプト例

```text
同じメモを踏まえて非機能要件を洗い出してください。
例えば、パフォーマンス/セキュリティ/可用性などの項目に分けて。
```

##### 出力例

```
[非機能要件]

1. パフォーマンス
   - 同時アクセス500ユーザー程度を想定
   - 応答時間: 2秒以内 (テキスト送信メイン)

2. 可用性
   - 24時間365日稼働
   - 大障害発生時の復旧目標は12時間
   - AWS等を利用し、低コストかつスケーラビリティを確保

3. セキュリティ
   - HTTPS通信の実装必須
   - ユーザーパスワードはハッシュ化して保存
   - 大規模の個人情報は扱わないが、一定の保護体制

4. 拡張性
   - 将来的に画像送信や多言語対応を追加できる構造
   - 広告機能をプラグイン化して導入可能に

5. 運用・メンテナンス
   - アプリのリリースサイクルは2〜3か月
   - 監視ログ: 基本的にCloudWatchなどでモニタ
```

---

## まとめ

- **ヒアリングメモ**がまとまっていなくても、**Copilot** に「ユースケースを作って」「機能要件にまとめて」「非機能要件を洗い出して」と**段階的に指示** → 整理がスムーズ  
- 上流工程で時間のかかる**要件分類**を**自動支援**し、**チームでの合意形成**を早める  
- その後、**優先度や難易度**をエンジニア側で調整し、**要件定義書**を確定 → 下流工程へ  

こうして**Copilot**を活用することで、**要件整理**が手軽になり、**ユースケース・機能要件・非機能要件**を迅速に取りまとめることができます。
