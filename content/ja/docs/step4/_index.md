## 🎯 課題の概要

これまで学んだ GitHub Copilot の高度機能（エージェントモード、カスタムインストラクション、カスタムチャットモード、エージェントワークフロー）を活用して、実際のプロジェクトに適用し、開発フローを体験する課題です。自分のプロジェクトで実際に手を動かすことで、実務レベルでの活用方法を習得することを目的とします。

---

## 📋 課題内容

### 🎯 お題
**自分のプロジェクトに GitHub Copilot のカスタマイズ機能を適用し、実際に活用できる状態を目指してください。**

#### 取り組み方針
以下の要素から、**着手できそう・プロジェクトに取り入れてみたいものを選んで**実装してください。順番は自由です。

#### 取り入れられるカスタマイズ要素

##### 📝 カスタムインストラクション
自分のプロジェクトに `.github/copilot-instructions.md` を設定
- プロジェクト概要と技術スタック
- 主要リポジトリの説明
- ビルド・テストコマンド
- 開発ガイドライン

##### 📁 パス別の個別ルール
パス別に個別のルールを `*.instructions.md` で設定
- `frontend.instructions.md` - フロントエンド固有のルール
- `test.instructions.md` - テスト作成のルール
- `document.instructions.md` - ドキュメント作成のルール

##### 👥 カスタムチャットモード
役割ごとのチャットモードを `.github/chatmodes/` に作成
- `architect.chatmode.md` - システム設計専門
- `frontend-engineer.chatmode.md` - フロントエンド実装専門
- `backend-engineer.chatmode.md` - バックエンド実装専門
- `reviewer.chatmode.md` - コードレビュー専門

##### 🔄 エージェントワークフロー
再利用可能なワークフローを `.github/prompts/` に作成
- `plan.prompt.md` - 実装計画策定
- `implement.prompt.md` - 実装実行
- `test.prompt.md` - テスト作成
- `review.prompt.md` - コードレビュー

##### ✅ 動作確認
作成したカスタマイズを実際に使ってみる
- プロンプト設計が適切か
- コンテキスト設定が正しいか
- 期待通りの動作をするか

## 💡 カスタマイズのヒント

### カスタムインストラクション例
```markdown
# プロジェクト概要
このプロジェクトは...

## 技術スタック
- フロントエンド: React + TypeScript
- バックエンド: Node.js + Express
- データベース: PostgreSQL

## 開発ガイドライン
- ESLintとPrettierの設定に従う
- テストは必須（Jest + React Testing Library）
- コミットメッセージはConventional Commits形式
```

### カスタムチャットモード例
- `architect.chatmode.md` - システム設計専門
- `frontend-developer.chatmode.md` - フロントエンド実装専門
- `backend-developer.chatmode.md` - バックエンド実装専門
- `reviewer.chatmode.md` - コードレビュー専門

### エージェントワークフロー例
- `plan.prompt.md` - 実装計画策定
- `implement.prompt.md` - 実装実行
- `test.prompt.md` - テスト作成
- `review.prompt.md` - コードレビュー

## ⏰ タイムスケジュール

| 時間 | 活動内容 | 詳細 |
|------|----------|------|
| **60分** | **フリータイム** | 個人で持ち込んだプロジェクトに対して課題を適用 |
| **30分** | **情報交換** | グループになって何をしたか発表 |
| **20分** | **全体発表** | グループの代表1名が実プロジェクトにどのように適用したか発表 |

## 📢 情報交換のポイント

情報交換タイムでは、以下について共有してください：

### 作成したカスタマイズ内容
- **カスタムインストラクション**：どのような内容を設定したか
- **カスタムチャットモード**：どのような役割分担で作成したか
- **エージェントワークフロー**：どのような開発フローを自動化したか

### 工夫した点と学び
- **効果的だった設定やプロンプト**
- **つまずいた点と解決方法**
- **予想以上に効果があった機能**
- **実際の開発での活用イメージ**

### 実際の適用例
- **どのようなプロジェクトに適用したか**
- **従来の開発フローとどう変わったか**
- **チーム開発での活用可能性**
