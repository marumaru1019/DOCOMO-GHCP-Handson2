要件定義や設計書を基に**「1つの機能要件」**をさらに **複数のタスク（チケット）**に細分化してエンジニアにアサインする作業は、プロジェクト管理の要です。**Copilot** を活用すると、**機能要件**から**タスク分割**を自動提案させ、**作業内容の指示**もあらかじめ盛り込めるため、**チケット作成**の初稿を素早く整備できます。

---

## シナリオ例

### 機能要件：ユーザーが画像付きメッセージを送信できる機能

現在はテキストメッセージのみ送信できるメッセージングアプリに、**画像を添付**して送る機能を追加したい。  

```text
- 要件概要:  
  1. ユーザーが画像を選択し、テキストと一緒に送信  
  2. 送信先の相手は画像を表示できる  
  3. 画像はサーバーにアップロード → メッセージデータベースと紐づけ → メッセージIDと関連付け  
  4. 通知にも「画像あり」の情報を載せる (将来的にはサムネイル対応も検討)
```

---

## :pen: プロンプト例

```text
下記の機能要件を、複数のタスク（チケット）に分割して作成してください。
各チケットには作業内容の指示も書き込んでほしいです。
想定工数も軽く推定して。

【機能要件】
「ユーザーが画像付きメッセージを送信できる機能を追加する」
1. ユーザーが画像を選択し、テキストと一緒に送信
2. サーバーで画像をアップロードし、メッセージDBと紐づけ
3. 受信側は画像を表示できる
4. 通知にも「画像あり」と付加情報を載せる

言語: Java(Spring Boot)/React
```

---

## :robot: 出力例

```
【タスクリスト】

1. [TICKET-IMG01] フロントエンド：画像アップロードUIの作成
   - 作業内容:
     - Reactのメッセージ送信フォームに「画像選択ボタン」を追加
     - 選択した画像のプレビュー表示
     - <input type="file" ...>を利用し、JavaScriptでファイルハンドリング
   - 想定工数: 2日

2. [TICKET-IMG02] サーバーサイド: 画像アップロードAPIの作成
   - 作業内容:
     - Spring Boot で POST /api/messages/upload 等のエンドポイント実装
     - MultipartFileを受け取り、ローカル or S3等に保存
     - 画像の保存パスやURLを返却
   - 想定工数: 3日

3. [TICKET-IMG03] メッセージDBスキーマ拡張
   - 作業内容:
     - "messages"テーブルに "imageUrl" カラムを追加
     - Flyway等のマイグレーションスクリプト作成
     - 既存メッセージ送信ロジックと連動させる
   - 想定工数: 1日

4. [TICKET-IMG04] フロントエンド: 画像表示対応
   - 作業内容:
     - 受信メッセージに imageUrl があれば <img>タグで表示
     - スタイル調整、サムネイル or フルサイズの切り替え検討
   - 想定工数: 2日

5. [TICKET-IMG05] 通知に「画像あり」情報を付与
   - 作業内容:
     - サーバーサイドで PUSH通知やWebSocket通知に "hasImage" フラグを追加
     - フロントで通知ポップアップに「画像付きメッセージ」を表示
   - 想定工数: 1日
```

> Copilot は**複数タスク**を**TICKET-IMGxx**などに分割し、それぞれに**作業内容の指示**(UI変更/DBマイグレーション等)と**想定工数**を添えてくれる可能性があります。

---

## ポイント

1. **機能要件**を**Copilot**に提示 → 「この要件を**複数チケット**に分解」「作業指示も書いて」「工数推定」  
2. 出力された**チケットリスト**を**JIRA, Redmine, GitHub Projects** などのツールに転記し、さらに**優先度**や**担当者**を追加  
3. **チームで検討**しつつ**現実的な工数**に修正 → **最終的なスプリント計画**へ落とし込み

---

## 練習

1. **さらに細かいサブタスク**  
   - 例えば「UIデザイン」「ユニットテスト」「ログイン必須チェック」などを増やす → Copilot に再依頼してみる  
2. **工数を再調整**  
   - 「工数を人日でなく、ポイント制(ストーリーポイント)で書いて」と頼んでみる  
3. **別の言語/フレームワーク**  
   - 「Node.js + Vue.jsで書く想定。チケットを作り直して」と依頼 → Copilot がどう反応するか観察

---

## まとめ

- **1つの機能要件** (例: 画像付きメッセージ機能) → Copilot が**複数チケット**に分割する  
- **作業内容**(UI変更, API作成, DBマイグレ)を**具体的**に書かせ → **初稿**を効率的に獲得  
- 最終的な**優先度, 担当者, 工数**はチームで調整 → Copilot の提案をたたき台に活用  

こうして**Copilot** に**チケット化作業**をサポートさせると、**プロジェクト全体のタスクリスト**作成がスムーズになり、**上流工程から下流工程**へスピーディに移行できます。
