GitHub Copilot を活用して、フロントエンドとバックエンドなど複数のリポジトリにまたがる開発を効率的に進める実践的なワークフローを学びます。

このセクションでは、マルチリポジトリ環境での AI 開発における以下の重要なテクニックを解説します:

* **マルチルートワークスペース管理** - VS Code で複数リポジトリを統合的に扱う
* **ワークスペースインデックス** - 大規模プロジェクトでの検索品質を向上させる
* **クロスリポジトリ参照** - フロントエンドとバックエンドの整合性を保つ
* **カスタムインストラクション** - プロジェクト固有のルールとベストプラクティスの定義
* **カスタムチャットモード** - リポジトリ固有の専門性を分離する
* **エージェントフロー** - 再利用可能なワークフローによる自律的なタスク実行
* **仕様駆動開発（spec-driven）** - specify→plan→implement→review サイクルの確立
* **メモリ機能** - プロジェクトコンテキストの永続化とドキュメント自動更新

これらのテクニックを習得することで、複数のリポジトリにまたがる複雑な開発タスクを、AI の支援を受けながら整合性を保ちつつ効率的に進めることができます。
