GitHub Copilot は、 **現在開いているファイル + 隣接タブ** のファイルを参考にコード提案を行う特性があります。ここでは **Java** の例を使って、 **「他のタブで開いているメソッドを呼び出す提案を勝手にしてくれる」** という隣接タブのメリットを示します。また、**ピン留め**を活用すると、よく使うファイルを常時参照させやすくなるため効率的です。

---

## :pen: 隣接タブでの例

### 1. 準備 – 隣のタブにヘルパークラスを開く

**ファイル①: `Helper.java`**  
```java
public class Helper {
    public static String greet(String name) {
        return "Hello, " + name + "!";
    }
}
```
このクラスを **タブ1** で開いておきます。

**ファイル②: `App.java`** (まだ中身が書きかけ)  
```java
public class App {
    public static void main(String[] args) {
        // Helper クラスを使って挨拶を表示
    }
}
```
このクラスを **タブ2** で開いて、途中まで書いて止めておきます。

### 2. Copilot が「隣タブのメソッド」を提案

**シナリオ**  
- 「Hello, ユーザー！」と表示したい時、Copilot が **`Helper.greet(...)`** を勝手に参照し、`main`メソッド内で呼び出す提案をしてくれるかもしれません。

**:robot: 出力例 (イメージ)**  
```java
public class App {
    public static void main(String[] args) {
        // Helper クラスを使って挨拶を表示
        String message = Helper.greet("Alice");
        System.out.println(message);
    }
}
```
Copilot は、**隣接タブで開かれた `Helper.java`** を参考に、「`greet` メソッドがある→呼び出せる」と判断し、自動で補完案を出してくれます。

## ピン留め (Pin Tab)

### 1. 概要

VS Code で特定のファイルを「ピン留め」しておくと、**重要ファイルが常に開いている状態**になり、Copilot もそれを参照しやすくなります。

**ピン留めの手順**:

- ファイルタブを右クリック → 「Pin Tab」を選択  
- タブ左にピンアイコンがつき、他のファイルを開いても閉じられない

### 2. 使い方の例

**例**: `Helper.java` をピン留め  
- このファイルを常時開いておくことで、Copilot が `App.java` でコードを書くときにも `Helper` の存在を安定的に参照  
- IDE上で **「ピン留め」** されているため、ユーザー自身もすぐに内容を確認・更新できる

---

## :memo: 練習

1. **呼び出し先ファイルを閉じたり開いたり**  
   - `Helper.java` を開いている状態と閉じた状態で、Copilot の提案がどのように変わるか観察してみる

2. **複数のヘルパーファイルをピン留め**  
   - いくつかユーティリティクラスがある場合、全部開いてピン留め → Copilot がそれらを利用しながら提案してくれるか検証

3. **不要ファイルを開かない**  
   - 大量の無関係なファイルを開くと、Copilot が文脈を混同する恐れあり → 必要ファイルだけ隣接タブ＆ピン留めで最適化

---

## まとめ

- **隣接タブ**:  
  - Copilot は「開いているファイル＋隣のタブ」でコード補完 → 必要なクラスやメソッドを参照させたい場合、該当ファイルを同時に開いておく  
- **ピン留め (Pin Tab)**:  
  - 重要ファイル (ヘルパー・インターフェイス・設定ファイルなど) をピン留めしておけば、常に Copilot の文脈として利用できる  
- **ポイント**:  
  - 「呼び出したいメソッドやクラスがあるファイルを隣接タブで開く」  
  - 「型定義やユーティリティをピン留め」→ Copilot が正確に参照  

こうした工夫により、Copilot が**隣接ファイルのメソッドを勝手に提案してくれる**ようになり、Javaの開発を一層スムーズに進められます。
