GitHub Copilot には、**チャットビュー** と **インラインチャット** という 2 つの対話的な機能があります。  
これらを使うと、単なるコード補完だけでなく、**「コードを書かせる」「コードの説明や修正をさせる」「テストを生成する」** といった操作を自然言語で依頼できます。ここでは、チャットビューとインラインチャットを分けて解説します。

---

## 1. チャットビュー

### 1-1. 概要
チャットビューは、**VS Code のサイドバーに表示される専用ペイン**で、Copilot と会話するためのインターフェイスです。過去のやり取り（会話履歴）を保持しながら、コード生成・リファクタ・ドキュメント作成などを頼むことができます。

### 1-2. 呼び出し方法
- **検索窓の右隣にあるチャットアイコン**をクリック  


### 1-3. 使い方の例
1. **コードを書かせる**  
   - 「Java で 1～10 の乱数を返すメソッドを書いて」と入力 → Copilot がコードを提示  
   - コードブロック横の「挿入」をクリックすればエディタへ直接貼り付け  
2. **コードの説明を依頼**  
   - 「このファイル内のメソッドが何をしているか説明して」  
   - さらに「パフォーマンス面はどうか？」などフォローアップ質問も可能  
3. **リファクタ／テスト生成**  
   - 「このメソッドをリファクタして」「テストコードを書いて」など  
   - 出力されたものをコピー or そのまま挿入し、エディタで編集

---

## 2. インラインチャット

### 2-1. 概要
インラインチャットは、**「特定のコードや行」を選択したうえで、キーボードショートカット**を使って Copilot に直接質問・命令できる機能です。  
チャットビューと異なり、**その瞬間の選択範囲**にフォーカスした対話を行うのに適しています。

### 2-2. 呼び出し方法
- **Ctrl + Shift + I (Windows/Linux)**  
- **Control + Command + I (Mac)**  

キーボードショートカットを押すと、エディタ画面上（またはポップアップ）に質問を入力するためのフィールドが現れます。

### 2-3. 使い方の例
1. **特定のコード選択 → ショートカット**  
   - 例えばメソッド本体を選択して、「この部分のロジックを分かりやすく書き直して」と入力  
   - Copilot がその行だけに焦点を当てた提案を返す  
2. **複数コマンド呼び出し**  
   - 「テストを生成して」「コメントを追加して」「ここだけリファクタして」など  
   - チャットビューに切り替えるほどでもない簡易操作の場合、インラインチャットが便利

### 2-4. Tips / 練習
- **ショートカットに慣れる**  
  - Ctrl+Shift+I / Control+Command+I を素早く押して、ちょっとした修正やテスト生成をサクッと依頼  
- **範囲選択を意識**  
  - 選択範囲に絞って質問すると、Copilot が不要な部分まで含めない提案をしてくれる  
- **チャットビューと併用**  
  - 大掛かりなやりとりはチャットビュー、ちょっとしたピンポイント修正はインラインチャット、と使い分けがオススメ

---

## :memo: まとめ

- **Copilot Chat** には **「チャットビュー」** と **「インラインチャット」** があり、それぞれ用途や呼び出し手順が異なる  
- **チャットビュー**  
  - 検索窓の右隣アイコンから起動  
  - 会話履歴を活かして大きな視点で質問やコマンドを繰り返せる  
- **インラインチャット**  
  - ショートカット (**Ctrl + Shift + I** / **Control + Command + I**)  
  - 選択したコードブロックにフォーカスしたやりとりに向いている  
- どちらも**テスト生成・リファクタ・コード説明**など、さまざまな命令を自然言語で行えるが、スコープと操作感が少し違う。プロジェクト規模や目的に合わせて使い分けると開発効率が上がる。
